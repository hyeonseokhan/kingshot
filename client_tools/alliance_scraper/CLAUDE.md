# Alliance Scraper — 작업 지침

> 이 디렉토리(client_tools/alliance_scraper)에서 작업할 때 항상 따라야 하는 지침이다.
> Claude 가 본 목적에서 벗어나지 않게 하기 위한 가드레일.

## 본 목적 (절대)

**RPA(ADB) + OCR 로 Kingshot 게임의 연맹원 4가지 데이터를 100% 정확히 추출한다.**

1. **랭크** — 게임 내 전투력 순위
2. **연맹원 ID** — 고유 식별자
3. **전투력** — 화면 표시값
4. **등급 (R1~R5)** — R5 는 연맹장 1명, 나머지는 다수

다른 모든 작업은 본 목적의 직접적 수단이거나 사용자가 명시적으로 요청한 경우에만 한다.

## 빠지지 말아야 할 함정 (이전에 빠진 적 있음)

- ❌ **phash 캐싱 / 시간 단축 시나리오** — 사용자가 "시간 단축은 무관, 정확도 100% 가 목표" 라 명시했음
- ❌ **avatar phash 의 hamming 거리 분석, dedup 정책 튜닝** 등 — 본 목적 외 우회로
- ❌ **다음 날 fresh scan 시 cache 매칭** — 사용자 무관
- ❌ **새 알고리즘 / 도구 도입을 사용자 확인 없이 시작** — 의사결정 점만 빠르게 짚고 사용자 신호 받기

이런 작업이 시작되려는 자기 자신을 발견하면 **즉시 멈추고 사용자에게 본 목적 재확인** 한다.

## 진행 원칙

1. **단위 테스트로 한 가지씩** — 4가지 데이터를 한꺼번에 검증하지 않는다. 한 데이터의 정확도를 100% 끌어올린 후 다음 데이터로.
2. **사용자가 게임 화면 보고 검증 가능한 출력** — 코드 결과와 사용자가 직접 본 게임 화면이 비교 가능해야 한다. (예: 추출된 1위 닉네임 vs 실제 1위 닉네임)
3. **짧은 호흡** — 한 번의 측정 → 결과 보고 → 다음 의사결정. 큰 스크립트로 한 번에 모든 걸 검증하지 않는다.
4. **TodoWrite 트래킹 필수** — 작업 단위마다 todo 업데이트. 본 목적 외 todo 가 끼어들면 의심한다.

## 우선순위 (순서대로)

1. **Step 1**: 랭크 정확도 — 첫 화면 + 스크롤 시
2. **Step 2**: 전투력 정확도 — 첫 화면 + 스크롤 시
3. **Step 3**: ID 정확도 — 프로필 진입 OCR
4. **Step 4**: 등급 (R1~R5) 인식 — 신규 구현 (이전 BACKLOG 코드 정리됨)

## 코드 위치

- 메인 진입: `scripts/scrape.py`
- 행 추출 (rank/power/avatar phash): `src/rows.py`
- 멤버 프로필 진입 + ID 추출: `src/scraper.py`
- DB 업로드: `scripts/upload_to_db.py`
- 등급 자산 (재사용): `assets/R1~R5.png`
- 단위 테스트: `scripts/test_step{N}_*.py` 명명

## 사용자 검증 패턴

각 단계의 단위 테스트는 다음 형태를 포함해야 한다:

```
=== 추출 결과 ===
rank | power | (가능하면 닉네임 추정)
  1  | 102M  | SsungBi    ← 사용자 게임 화면 1위 닉네임과 비교 가능
  2  | 99M   | Raducanu
  ...

=== 정확도 ===
- 추출 성공: N/M (X%)
- 실패 케이스: rank 39, 90, ... (사용자에게 명시)
```

사용자가 결과를 보고 "맞다/틀리다" 직접 판단할 수 있어야 한다.
