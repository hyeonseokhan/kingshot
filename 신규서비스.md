건축 버프 최적화 웹 도구 개발 가이드

1. 프로젝트 개요

이 웹 도구는 모바일 게임 내 건축 시스템에서 사용자가 보유한 건축 관련 버프와 현재 건축 큐 상황을 입력하면, 다음 행동을 추천하는 계산기이다.

핵심 목적은 다음과 같다.

- 지금 바로 건축을 시작하는 것이 좋은지 판단한다.
- 총리대신 버프를 기다렸다가 건축하는 것이 좋은지 판단한다.
- 최대 2개의 동시 건축 큐를 고려하여 최적의 건축 순서를 추천한다.
- 법령, 펫, 총리대신 버프를 조합하여 가장 효율적인 건축 시작 타이밍을 계산한다.
- 보유한 건축 가속권을 사용할 경우의 최적화 가능성도 함께 계산한다.

⸻

2. 게임 시스템 전제

2.1 동시 건축 가능 수

- 건축물은 최대 2개까지 동시에 건설할 수 있다.
- 사용자는 현재 건축 큐 1, 건축 큐 2의 상태를 각각 입력할 수 있어야 한다.

  2.2 버프 적용 타이밍

- 모든 건축 관련 버프는 건축을 시작하는 순간의 상태만 최종 반영된다.
- 건축 시작 후 버프를 받아도 이미 진행 중인 건축물에는 효과가 적용되지 않는다.
- 따라서 이 도구는 “언제 건축을 시작할 것인가”를 판단하는 도구이다.

  2.3 건축 시간 계산 방식

건축 시간은 다음 공식으로 계산한다.

최종 건축 시간 = 기본 건축 시간 ÷ (1 + 건설 속도 버프 합계) × (1 - 건축 시간 단축 버프)

예시:

기본 건축 시간: 29일 2시간 52분
건설 속도 버프: 76.6%
건축 시간 단축 버프: 20%
최종 건축 시간 = 기본시간 ÷ 1.766 × 0.8

⸻

3. 버프 종류

3.1 기본 건설 속도

- 사용자의 연구, 스탯 등에 따라 달라지는 기본 건설 속도이다.
- 모든 사용자가 다르므로 반드시 직접 입력받아야 한다.

입력 예시:

54.6%

⸻

3.2 법령: 건설 가속

- 재사용 대기시간: 24시간
- 법령 사용 후 5분 이내에 건설을 시작하는 모든 건축물의 업그레이드 시간이 20% 단축된다.
- 건설 속도 증가가 아니라 **최종 시간 단축 20%**로 계산한다.

계산 반영:

건축 시간 단축 버프 = 20%

⸻

3.3 펫: 회색늑대 스킬

- 재사용 대기시간: 23시간
- 스킬 사용 후 5분 동안 건설 속도가 증가한다.
- 펫 레벨에 따라 증가량이 다르다.

가능 수치:

5% / 7% / 9% / 12% / 15%

계산 반영:

건설 속도 버프 합계에 더한다.

예시:

기본 건설 속도 54.6% + 펫 12% = 66.6%

⸻

3.4 임명: 총리대신

- 건설 속도를 10% 증가시킨다.
- 신청 후 순차적으로 부여받는 권한이므로 정확한 대기 시간은 시스템이 알 수 없다.
- 사용자가 직접 “총리대신까지 남은 시간”을 입력해야 한다.

계산 반영:

건설 속도 버프 합계에 +10% 추가

예시:

기본 건설 속도 54.6% + 펫 12% + 총리대신 10% = 76.6%

⸻

- 비어 있음

비어 있을 경우:

각 건축물 입력값:

건축물 이름: 선택 입력
기본 건축 시간: 필수 입력
우선순위: 선택 입력

예시:

건물 A: 도시 센터 27레벨 / 29일 2시간 52분
건물 B: 병영 26레벨 / 7일 4시간 10분

5. 시간 입력 방식

시간은 내부적으로 초 단위로 변환하여 계산한다.

사용자 입력 UI는 다음처럼 분리한다.

일 / 시간 / 분 / 초

예시:

29일 2시간 52분 0초

내부 변환:

초 = 일 × 86400 + 시간 × 3600 + 분 × 60 + 초

⸻

6. 기본 계산 함수

6.1 건축 시간 계산 함수

function calculateBuildTime(
baseSeconds: number,
speedBuffPercent: number,
timeReductionPercent: number
): number {
const speedMultiplier = 1 + speedBuffPercent / 100;
const reductionMultiplier = 1 - timeReductionPercent / 100;
return baseSeconds / speedMultiplier \* reductionMultiplier;
}

⸻

6.2 즉시 건축 시간

총리대신을 제외하고 현재 사용 가능한 버프만 적용한다.

즉시 건축 시간 = 기본 건축 시간 ÷ (1 + 현재 건설 속도 버프 합계) × (1 - 시간 단축 버프)

현재 건설 속도 버프 합계:

기본 건설 속도 + 펫 버프

⸻

6.3 총리대신 대기 후 건축 시간

대기 후 총 소요 시간 = 총리대신 대기 시간 + 기본 건축 시간 ÷ (1 + 현재 건설 속도 버프 합계 + 10%) × (1 - 시간 단축 버프)

⸻

6.4 이득 시간

이득 시간 = 즉시 건축 완료까지 걸리는 시간 - 대기 후 완료까지 걸리는 시간

판정:

이득 시간 > 0 → 기다리는 것이 이득
이득 시간 = 0 → 동일
이득 시간 < 0 → 바로 건축하는 것이 이득

⸻

7. 손익분기 계산

손익분기 대기 시간은 총리대신을 기다려도 되는 최대 시간이다.

손익분기 대기 시간 = 총리대신 없이 계산한 건축 시간 - 총리대신 포함 계산한 건축 시간

예시:

총리대신 없이 건축 시간: 13일 23시간 35분
총리대신 포함 건축 시간: 13일 4시간 35분
손익분기 대기 시간: 19시간

출력 예시:

이 건축물은 총리대신을 최대 19시간까지 기다려도 이득입니다.
현재 총리대신까지 남은 시간이 11시간이므로 기다리는 것이 유리합니다.

⸻

8. 2개 건축 큐 최적화 로직

이 도구의 핵심은 최대 2개의 동시 건축 큐를 고려하는 것이다.

8.1 기본 의사결정

사용자는 다음 상황 중 하나에 해당한다.

1. 큐 2개가 모두 비어 있음
2. 큐 1개만 비어 있음
3. 큐 2개가 모두 건축 중
4. 큐가 곧 비지만 총리대신 대기 시간이 남아 있음

도구는 각 큐가 비는 시간과 총리대신 획득 시간을 비교해야 한다.

⸻

8.2 핵심 전략

긴 건축물일수록 총리대신 효과가 크다.

따라서 다음 원칙을 따른다.

짧은 건축물 → 총리대신 없이 시작해도 손해가 작다.
긴 건축물 → 총리대신, 펫, 법령을 모두 적용하는 것이 유리하다.

⸻

8.3 추천 패턴

패턴 A: 큐 1개가 곧 비고 총리대신은 아직 멀었음

예시:

큐 1: 2시간 뒤 완료
큐 2: 건축 중
총리대신: 13시간 뒤
다음 건축물 A: 3일
다음 건축물 B: 29일

추천:

건축물 A는 큐가 비는 즉시 시작하세요.
건축물 B는 총리대신을 받은 후 법령 + 펫 + 총리대신을 모두 적용하여 시작하세요.

⸻

패턴 B: 긴 건축물 하나만 있고 총리대신 대기 시간이 손익분기보다 짧음

추천:

총리대신을 기다린 후 건축하세요.

⸻

패턴 C: 짧은 건축물이고 총리대신 대기 시간이 손익분기보다 김

추천:

기다리지 말고 바로 건축하세요.

⸻

패턴 D: 큐 2개가 모두 비어 있고 건축 후보가 2개 이상 있음

추천 기준:

1. 각 건축물의 손익분기 대기 시간을 계산한다.
2. 총리대신 대기 시간보다 손익분기 시간이 긴 건축물은 총리대신 대기 후보로 분류한다.
3. 후보 중 기본 건축 시간이 가장 긴 건축물에 총리대신을 우선 적용한다.
4. 나머지 짧은 건축물은 즉시 건축 후보로 추천한다.

예시 출력:

추천 전략:

1. 병영은 지금 바로 시작하세요.
2. 도시 센터는 총리대신을 받은 후 법령 + 펫 + 총리대신을 모두 적용하세요.
   예상 이득:
   도시 센터에 총리대신을 적용하면 약 8시간 이득입니다.

⸻

9. 가속권 계산 로직

9.1 가속권 기본 개념

사용자는 건축 가속권을 보유하고 있을 수 있다.

입력 예시:

5일 12시간 14분

도구는 가속권을 사용했을 때 다음을 계산해야 한다.

- 현재 진행 중인 건축물을 가속해서 큐를 비우는 것이 이득인지
- 총리대신 타이밍에 맞추기 위해 가속권을 일부 사용하는 것이 이득인지
- 가속권을 사용하지 않고 기다리는 것이 나은지

⸻

9.2 가속권 사용 추천 원칙

가속권은 무조건 사용 추천하지 않는다.

다음 경우에만 추천한다.

가속권 사용으로 인해 긴 건축물에 총리대신 + 법령 + 펫을 적용할 수 있고,
그로 인해 절약되는 시간이 사용한 가속권 시간보다 큰 경우

⸻

9.3 가속권 판단식

가속권 사용 이득 = 버프 최적화로 절약되는 시간 - 사용한 가속권 시간

판정:

가속권 사용 이득 > 0 → 가속권 사용 추천
가속권 사용 이득 <= 0 → 가속권 사용 비추천

⸻

9.4 출력 예시

현재 큐가 4시간 뒤에 비지만, 총리대신은 2시간 뒤에 받을 수 있습니다.
건축 가속권 2시간을 사용해 큐를 먼저 비우면,
도시 센터에 총리대신 버프를 적용할 수 있습니다.
예상 절약 시간: 19시간
사용 가속권: 2시간
순이득: 17시간
추천: 가속권 2시간 사용 후 도시 센터를 시작하세요.

12. 결과 출력 UX

12.1 최종 추천 카드

예시:

추천: 기다렸다가 도시 센터를 시작하세요.
총리대신까지 남은 시간은 11시간입니다.
도시 센터는 총리대신을 기다렸을 때 약 8시간 이득입니다.
짧은 건물은 지금 바로 시작하고,

15. 중요한 구현 주의사항

15.1 퍼센트 계산

사용자가 54.6을 입력하면 내부에서는 0.546이 아니라 54.6으로 보관하고, 계산 시 / 100 처리한다.

const speedMultiplier = 1 + speedBuffPercent / 100;

⸻

15.2 시간 반올림

게임 표시 시간과 1~2초 차이가 날 수 있다.

출력 시에는 초 단위까지 계산하되, 기본 출력은 다음 방식이 좋다.

13일 4시간 35분

상세 보기에서만 초를 표시한다.

⸻

15.3 법령과 펫의 5분 제한

법령과 펫은 사용 후 5분 이내 시작한 건축물에 적용된다.

따라서 추천 결과에서 반드시 다음 문구를 포함한다.

법령과 펫은 사용 후 5분 이내에 건축을 시작해야 적용됩니다.

16. 핵심 사용자 시나리오

시나리오 1: 긴 건물을 총리대신까지 기다릴지 판단

입력:

기본 건설 속도: 54.6%
펫: 12%
법령: 사용
총리대신까지 남은 시간: 11시간
건물: 도시 센터 / 29일 2시간 52분

출력:

기다리는 것이 이득입니다.
예상 이득은 약 8시간입니다.

⸻

시나리오 2: 짧은 건물과 긴 건물이 함께 있을 때

입력:

총리대신까지 남은 시간: 11시간
건물 A: 3일
건물 B: 29일

출력:

건물 A는 지금 바로 시작하세요.
건물 B는 총리대신을 받은 후 법령 + 펫 + 총리대신을 적용하세요.

⸻

시나리오 3: 가속권 사용 여부 판단

입력:

큐 1 남은 시간: 13시간
총리대신까지 남은 시간: 11시간
보유 가속권: 5일 12시간
긴 건물: 29일

출력:

큐를 2시간 가속하면 총리대신 타이밍에 맞춰 긴 건물을 시작할 수 있습니다.
예상 순이득이 양수라면 가속권 사용을 추천합니다.

17. 최종 목표

이 도구는 단순 계산기가 아니라, 사용자가 아래 질문에 바로 답을 얻도록 만드는 것이 목표이다.

지금 지을까?
기다릴까?
짧은 건물을 먼저 지을까?
긴 건물에 버프를 몰아줄까?
가속권을 지금 써도 될까?

따라서 결과는 수식 중심보다 행동 추천 중심으로 제공해야 한다.

최종 출력은 다음처럼 명확해야 한다.

추천: 짧은 건물은 지금 시작하고, 긴 건물은 총리대신을 받은 후 시작하세요.
이유:
긴 건물은 총리대신 적용 시 약 19시간이 단축됩니다.
현재 총리대신까지 남은 시간은 11시간이므로 약 8시간 이득입니다.
